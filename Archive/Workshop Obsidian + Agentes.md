# Workshop Master Doc (Detailed)

## Working title options (Luma)

- Obsidian x AI Agents: Build Your Second Brain
- Second Brain Lab: Obsidian, Agents, and Collective Intelligence
- From Personal Knowledge to a Shared Mind: Obsidian + Agents
- Build a Living Knowledge System (Workshop)

## Core concept

This workshop demonstrates a practical path:

1. Build a personal second brain in Obsidian.
2. Add AI agents on top of that knowledge base.
3. Expand from individual cognition to collaborative intelligence ("egregore of minds").

The narrative connects naturally with last year's Ipê Mind Tree work:

- Ipê Mind Tree introduced collective memory and shared navigation.
- This workshop shows the operational layer anyone can build personally first.
- Then it shows how these same primitives can scale to community-level intelligence.

## Why this matters

Most people use note-taking tools as storage.
This workshop reframes the system as:

- structured memory,
- context-aware assistant layer,
- composable workflows.

The goal is not only productivity.
The deeper aim is to help people build sovereign cognitive infrastructure.

## Intended audience

- Builders, founders, researchers, creators
- People curious about Obsidian, AI workflows, and knowledge systems
- Both technical and non-technical participants (no coding required to benefit)

## Learning outcomes

By the end of the workshop, participants should:

- Understand the architecture of a second brain beyond folders and tags.
- See how agent behavior changes with context, rules, and memory.
- Learn a concrete workflow: idea -> map -> note -> agent-assisted refinement.
- Understand the bridge from personal systems to community knowledge systems.
- Leave with a realistic first implementation path.

## Event format and timing (target: ~2h)

### Block 1 (0-15 min) - Framing

- Why second brains now
- Difference between note archive and living knowledge system
- The "personal first, collective later" principle

### Block 2 (15-45 min) - Personal knowledge base foundation

- Obsidian primitives: notes, links, maps/canvas, project hubs
- Minimal architecture for a robust personal system
- Example of navigation and retrieval

### Block 3 (45-75 min) - Agent layer

- What agents are in practical terms (context + rules + role)
- Live demonstration of agent-assisted work
- Why context quality matters more than model hype

### Block 4 (75-105 min) - From self to network

- Continuity with Ipê Mind Tree
- Patterns for shared knowledge and collaborative intelligence
- Risks and guardrails: fragmentation, hallucination, governance

### Block 5 (105-120 min) - Action plan and Q&A

- First-week implementation checklist
- Recommended starter stack
- Open Q&A

## Demo script ideas (practical sequence)

1. Start with a raw note (problem/idea).
2. Connect it to a canvas or concept map.
3. Convert map insights into structured notes.
4. Invoke an agent with explicit context and objective.
5. Show iterative refinement loop.
6. Show how this can become shareable/team-readable.

## Narrative spine for the talk

### Act 1 - Personal clarity

"Before collective intelligence, you need cognitive clarity at the individual level."

### Act 2 - Augmented cognition

"Agents are not magic; they are behavior over context."

### Act 3 - Shared intelligence

"When many people maintain interoperable memory systems, collaboration quality compounds."

## Suggested speaking tone

- High energy, grounded in practical examples
- Visionary but concrete
- Encourage experimentation, not rigid templates
- Emphasize participant autonomy ("build your own version")
